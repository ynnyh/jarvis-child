# 视觉设计升级 v3（满屏童趣）

> 交互已到位，但页面本身不够"小朋友一眼喜欢"。本轮做纯视觉/展示层升级，对标洪恩的童趣视觉。
> 不改功能、不改数据、不改后端。

## 一、现状诊断（为什么不够童趣）

1. **字体没加载**：声明了圆体但没引入 web font，实际全是系统微软雅黑，不圆润。
2. **背景平淡**：只有 5 处渐变，基本平铺色，无场景感、无云朵星星气泡。
3. **缺惊喜时刻**：无撒花/粒子/庆祝大动画。
4. **按钮偏扁平**：有圆角投影，但缺洪恩那种"果冻糖果感"（高光+厚底边+按下回弹下沉）。
5. **小墨露脸少、反应少**。

## 二、字体方案（UI 圆体 + 汉字楷体）

- **拉丁/数字/拼音**：自托管圆体 web font（Baloo 2，只含拉丁字符，体积小），存 `public/fonts/`。
- **学习用汉字**（大字展示：认读/字理/情境）：用 `楷体`（KaiTi/STKaiti 系统字体），字形标准、适合识字，**不用圆体**（圆体会让字形变形、误导书写）。
- **中文 UI 文字**（标题/按钮/提示）：保持系统圆润无衬线栈（PingFang/微软雅黑），不额外加大字体文件。
- 新增令牌：`--font-kai`（楷体，给汉字）、`--font-ui`（含 Baloo 2 的圆体栈）。

## 三、满屏童趣视觉系统

**果冻糖果按钮/卡片**（升级 tokens + 组件样式）：
- 渐变填充 + 顶部内高光（像糖果反光）
- 厚彩色底边（skeuomorphic，按下时下沉 + 回弹）
- 更大圆角、更软阴影

**场景化背景**（新组件 `PlayfulBackground`）：
- 每屏一个主题渐变天空
- 漂浮装饰：云朵、星星、气泡、太阳，用 SVG/emoji 绝对定位 + CSS 慢速漂移动画
- 按页面主题换色（5 世界各自的天空色）

**撒花/粒子庆祝**（新组件 `Confetti`）：
- 触发点：学完一个字、整课通关、开宝箱、宠物升级
- 彩色纸片 + 星星粒子从中心迸发飞散，配小墨欢呼

**小墨情绪反应**（复用 Xiaomo，新增出场时机）：
- 游戏/检查答对：小墨从边角弹出欢呼（cheer）
- 答错：小墨鼓励表情（encourage）
- 更多页面露脸陪伴

## 四、落地清单

**新增组件：**
- `Confetti.jsx` —— 撒花粒子（framer-motion 驱动，纯 CSS 元素，无素材）
- `PlayfulBackground.jsx` —— 漂浮装饰背景（云/星/泡，主题色可配）
- `MascotReaction.jsx` —— 答对/答错时小墨弹出反应

**字体：**
- 下载 Baloo 2 拉丁 woff2 → `public/fonts/`，`@font-face` 引入
- tokens 加 `--font-kai` / `--font-ui`；汉字大字类应用楷体

**样式升级（global.css / tokens.css）：**
- 果冻按钮 `.ui-btn`、世界卡片 `.world-card`、关卡节点 `.map-node`、各卡片
- 各页接入 `PlayfulBackground`

**接入惊喜时刻：**
- `StarReward` / `Chest` / 宠物升级 → 加 `Confetti`
- `GamePlay` / `LearnFlow` 的 check → 加 `MascotReaction`

## 五、原则（别翻车）

- **学习内容优先清晰**：正在学的大字放在足够干净的底上，装饰不抢戏、不遮挡。
- **性能**：装饰用 CSS transform 动画（GPU），粒子数量克制，低端 pad 不卡。
- **可访问性**：动画不影响点击热区；保留大按钮、大字。

## 六、不做

- 不引入图片/插画素材（维持纯 SVG/CSS/emoji）
- 不改任何功能逻辑与数据
