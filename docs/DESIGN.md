# 宝宝识字 · 完整产品设计 v1

> 从 demo 升级为对标洪恩识字「架构与体验密度」的识字产品。自用、http、暂不做安全加固（JWT/密码哈希代码照常写，但不强制 https）。

## 一、设计总览

**目标**：完整学习路径 + 复习闭环 + 养成激励，对标洪恩的结构与体验密度，而非美术投入。

**「不简陋」靠什么**：
1. 统一的视觉设计系统（色彩/圆角/字体/间距规范）+ 自绘 SVG 吉祥物。
2. 动效（framer-motion）：转场、翻卡、奖励弹跳、宠物动作。
3. 学习深度：象形演变、间隔复习、多游戏、养成。

**诚实边界**：
- 吉祥物/场景为自绘 SVG，质感到「清爽可爱」，达不到专业插画级。
- 象形演变动画仅约 20-30 个象形字（日月水火山人口目等），非通用功能。
- 300 字拼音/组词自动生成，多音字与组词准确性需人工抽查。

## 二、视觉设计系统

- 主色：暖黄 `#FFD166`、珊瑚粉、薄荷绿、天蓝、葡萄紫（五大主题各一色）。
- 大圆角（20-28px）、厚投影、大按钮（触屏友好）、无衬线圆体字。
- 强语音引导 + 音效（点击/正确/错误/翻页/开宝箱）。
- **吉祥物 = 宠物（一物两用）**：熊猫幼崽「小墨」，纯 SVG，3-4 个成长阶段（幼崽→小童→少年）。平时陪伴 + 表情反馈，同时是养成对象。
- SVG 组件库：吉祥物各表情/姿态、场景元素、图标、装饰；CSS/framer-motion 驱动。emoji 继续作字的联想图。

## 三、内容体系（300 字 / 30 课）

5 大主题 → 每主题 6 课 → 每课约 10 字，难度由易到难。

| 主题 | 课数 | 示例 |
|---|---|---|
| 🔢 我的第一课（数字/方位/大小） | 6 | 一二三…十、上下左右、大小多少 |
| 🌳 大自然（天地日月/水火山石/动植物） | 6 | 日月水火山、云雨风雪、花草树木 |
| 👀 我自己（身体/五官/动作） | 6 | 人口手足目耳、口鼻牙、看听说走跑 |
| 🐱 万物（动物/食物/物品） | 6 | 猫狗鸟鱼、米饭菜果、门窗车灯 |
| ❤️ 我的生活（家人/颜色/常用） | 6 | 爸妈哥姐、红黄蓝白、你我他好 |

**单字数据模型**：
```js
{
  char: '日', pinyin: 'rì', strokes: 4,
  radical: '日', components: ['日'],   // hanzi_chaizi 拆字
  emoji: '☀️', hint: '太阳圆圆的，就是日。',
  words: [{ word: '日出', emoji: '🌅' }, { word: '生日' }],
  sentence: '今天是我的生日。',          // 供分级阅读
  pictograph: ['🌞→◐→日'],              // 仅象形字，其余 null
  themeId, lessonId
}
```

- 内容来源：基于常用字表 + 3-6 岁认知，手工编排分课/组词/释义/emoji。
- 音频：沿用 edge-tts 流水线（约 300 字 + 700 组词/例句 ≈ 1000 条 mp3，幂等生成入库）。
- 拆字：集成 `hanzi_chaizi`（Apache-2.0）。

## 四、功能设计

1. **学习路径地图**：蜿蜒小路串 30 关，逐课解锁，节点显示星级/锁状态，小墨站当前进度，视差 SVG 背景。
2. **单字学习闭环（深化）**：情境引入 → 象形演变（仅象形字）→ 认读（大字+拼音+组词配图，点读）→ 书写（hanzi-writer 笔顺+描红）。整课结算 + 金币 + 解锁下一关。
3. **巩固小游戏（6 种）**：听音选字（有）、看图选字、看字选图、连连看、笔顺描红（有库）、找朋友。
4. **间隔复习（Leitner 盒子法）**：字有掌握等级（盒 1-5），答对升级答错回 1；间隔 1→2→4→7→15 天；首页每日「今日复习」任务。
5. **奖励养成**：金币（学习/游戏/复习赚）、宝箱（里程碑/打卡开）、宠物小墨（喂养→升级进化）、每日打卡。
6. **家长中心**：算术门禁；学习报告（已学字数/掌握分布/时长/打卡/复习提醒）；多孩子档案。

## 五、技术架构

**前端**：React 18 + Vite + react-router-dom（现有）；新增 `zustand`（全局状态）、`framer-motion`（动效）；hanzi-writer、edge-tts 音频（现有模式）；自绘 SVG 组件库。字库/笔画/音频打包进前端（离线），后端只存用户进度/经济数据。

**后端（沿用服务器现成栈）**：FastAPI + SQLAlchemy + Alembic；MySQL 在现有 `spark-mysql` 新建独立库 `jarvis_child`；Redis 用现有 `spark-redis`；JWT + bcrypt。家长账号 + 多孩子档案。

**数据表**：`users`（家长）、`profiles`（孩子）、`progress`（每字掌握度+复习时间）、`economy`（金币/宠物）、`daily_logs`（打卡/时长）。

**API**：`/auth/*`、`/profiles`、`/progress/sync`、`/review/queue`、`/economy/*`、`/report`。

**同步**：本地优先（localStorage/IndexedDB）+ 后台同步，离线可玩，联网自动同步。

**部署**：前端现有 nginx 容器 712（不变）；后端新建 FastAPI 容器，对外端口 713；nginx `/api` 反代后端；MySQL 新建库；密钥环境变量注入。

**安全（自用暂缓）**：http 自用、不强制 https、暂不做限流/证书。代码层面 JWT + bcrypt 照常实现。上线公网前需补 https、限流、隐私最小化。

## 七、已定决策

- 后端对外端口：713。
- https：自用暂不做，先 http，别用真实密码。
- 吉祥物：熊猫幼崽「小墨」。
- 象形演变：仅约 20-30 个象形字。
- 内容审核：300 字组词/拼音生成后人工抽查（尤其多音字）。
