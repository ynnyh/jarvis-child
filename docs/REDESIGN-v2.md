# 交互重设计 v2

> 一阶段跑通了闭环，但交互密度、"好玩度"、关卡解锁体验不如洪恩。本轮重设计地图与单字学习。
> 对标依据：洪恩识字官方描述（字源字形字义趣味互动、偏旁部首专项、遵循记忆曲线、原创绘本用已学字编故事）。

## 一、核心诊断

**"简陋"的真正原因：数据够，但没用足。**
- 300 个例句全部存在，但学习流程**完全没用**。
- 295 个字有部件拆分（妈=女+马），但只有 28 个象形字讲了"字理"，其余 267 字学的时候**没有任何"字怎么来的"讲解**。

**"关卡解锁奇怪"的原因：** 30 关扁平锯齿排列，像长列表，没有世界结构、没有里程碑、没有冒险感。

## 二、地图重设计（五个主题冒险世界）

两级结构：
1. **世界选择页**（新首页）：5 个主题「小世界」大卡片
   - 数字乐园 🔢 / 自然森林 🌳 / 身体乐园 👀 / 动物王国 🐱 / 温馨小家 ❤️
   - 每个世界：场景化背景（CSS 渐变 + emoji 装饰，非手绘）、主题色、进度（X/6 关）、锁/开状态
   - 世界解锁：世界 N 需世界 N-1 的 6 关都 ≥1 星才开
2. **世界内路径页**：进入某世界后，6 个关卡站点串成蜿蜒小路，小墨沿路站在当前关
   - 关卡解锁：世界内 6 关顺序解锁
   - 关卡状态：🔒锁定 / ▶️当前(脉冲) / ⭐星级
   - **世界通关庆祝**：6 关全 3 星 → 奖杯 + 宝箱奖励，给明确里程碑

这样"冒险感"来自：场景背景 + 行走的小墨 + 世界里程碑，而非手绘素材。

## 三、单字学习重设计（全部加深）

现流程：情境 →(象形,仅28字)→ 认读 → 书写。**浅**。

新流程（每字）：
1. **情境导入 story**：大 emoji 场景 + 字带揭示动画"活起来"（缩放/弹跳/发光入场，统一动效，非逐字手绘）+ 小墨语音引出
2. **字理 etymology（扩到全 300 字）**：
   - 象形字（28）：保留演变动画
   - 其余（267）：**用部件拆分讲字**（妈=女+马），部件逐个飞入组成字
   - 配一个**常用部首含义小词典**（约 40-50 个：氵=和水有关、女=和女性有关、口=和嘴有关、扌=和手有关…），有则显示部首含义，增强"字理"教育性
3. **认读 read**：大字 + 拼音 + 点读（保留）
4. **组词 words**：每个词可点读 + 配图（用足 223 个带 emoji 的词）
5. **例句情境 sentence（新）**：展示例句，目标字高亮，整句朗读 + 点字复读（用足 300 例句）
6. **书写 write**：hanzi-writer 描红（保留）
7. **即时检查 check（新）**：写完一道快速小题（3 选 1 认字），答对给星，即时巩固

为控节奏：story/字理/例句/检查都是"快步骤"（自动播放或一两次点击即过），书写是主活动。

## 四、技术方案

- **无需改后端/数据模型**：世界解锁、通关庆祝都从现有 lessons 星级派生。
- **新增数据**：`src/data/radicals.js` 常用部首含义小词典（人工编写 ~50 条）。
- **新增组件**：
  - `WorldSelect`（世界选择首页，替代现 MapHome 首页角色）
  - `WorldPath`（世界内关卡路径，复用现 MapHome 的路径逻辑）
  - `Etymology`（字理讲解，象形走 Pictograph，其余走部件飞入）
  - 单字流程 `LearnFlow` 扩成 7 步
- **改动**：路由加 `/world/:themeId`；首页改为世界选择；小墨走位动画。

## 五、开发步骤

1. 部首含义词典 `radicals.js`（~50 条）
2. `Etymology` 组件（象形/部件两种模式）
3. `LearnFlow` 重构为 7 步（含例句、即时检查）
4. 世界选择首页 `WorldSelect` + 世界路径页 `WorldPath`
5. 世界解锁 + 通关庆祝逻辑
6. 样式（世界场景、路径、字理飞入、例句高亮）
7. 构建验证 + 部署 712

## 六、暂不做（避免范围膨胀，后续可加）

- 原创绘本（洪恩有 130 本"用已学字编故事"）—— 数据量大，二期单独做
- 成语板块、拼音板块 —— 超出识字范围
- 真人配音 —— 现用 edge-tts 够用
